# FOOD DELIVERY API
 
REST API для сервиса доставки еды с поддержкой полного цикла заказа. 

## Основные возможности: 

## Пользователи: 
- Регистрация и управление профилем

## Рестораны и меню:
- Создание и управление ресторанами 
- Поддержка динамического меню
- Расчет среднего рейтинга на основе отзывов

## Корзина: 
- Добавление блюд из 1 ресторана 
- Учет времени работы ресторана
- Временная заглушка пользователя: возвращает ID = 1
- Расчет предварительного ЕТА доставки 

## Заказы:
- Создание заказа на основе корзины 
- Проверка доступности блюд и бизнес-правил заказов 

## Курьеры:
- Регистрация курьеров 
- Назначение заказов с проверкой рейтинга курьеров 

## Отзывы:
- Анти-спам защита комментариев 
- Временная заглушка пользователя: возвращает ID = 1

## Уведомления:
- Push-уведомления на основе статуса заказа (в данный момент запись идет в логи)
- Отложенная отправка уведомлений

## Технологический стек 

- Java 17
- Spring Boot 3.x (Web, Validation, Data Jpa)
- H2 database
- JUnit, Mockito
- Lombok
- MapStruct
- Swagger 

##  Установка и запуск

### Локальный запуск

1. Клонируйте репозиторий:
   git clone https://github.com/DariaB14/FoodDelivery.git
3. Соберите проект:
   mvn clean install 
4. Запустите приложение:
   mvn spring-boot:run

## API документация

После запуска приложения документация доступна по адресу:

Swagger UI: http://localhost:8080/swagger-ui.html

## Планы развития:
1. Реализовать поддержку комбинированных заказов и сплит-платежей;
2. Расширить систему уведомлений с интеграцией push, SMS, email каналов; 
3. Интегрировать геотрекинг курьеров для более точного расчета доставки; 
4. Реализовать систему версионности данных пользователя;
5. Разработать систему персонализации на основе действий пользователя, его предпочтений и истории заказов; 
6. Переход на PostgreSQL с использованием Liquibase для миграций; 
7. Реализовать файловую поддержку для загрузки изображений; 
8. Внедрить Spring Security с ролевой моделью доступа; 
9. Контейнеризация приложения; 
10. Переход на микросервисную архитектуру для обеспечения масштабируемости приложения.
